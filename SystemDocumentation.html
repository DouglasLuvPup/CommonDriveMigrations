Executive Summary

Automated file share migration system that migrates on-premises Common Drive data to Microsoft Teams/SharePoint Online. The system uses distributed processing across 6 servers (18 parallel instances) with app-only authentication for fully unattended operation.

CategoryCapabilities
Scheduling Timezone-aware windows (EST, CST, MST, PST, AKST, HST) — migrations run 5 PM - 4 AM local time on weekdays • Large migrations (≥10GB) restricted to weekends (Fri 5PM → Mon 4AM) and federal holidays • ANYTIME option bypasses all restrictions for 24/7 processing
Priority Queuing Priority 1-10 scale (lower = higher priority, default 5) • FIFO within same priority level • QueuedAt timestamp for fair ordering • Items processed in Priority → QueuedAt order
Data Management 3/5/7 year data retention analysis • Automatic storage quota validation before migration • Post-migration source cleanup (read-only + file deletion + empty folder cleanup)
Automation Automatic Team/Channel resolution via Graph API • Fully unattended app-only authentication (certificate-based) • 36 Azure apps for distributed throttling • 18 parallel SPMT workers
Notifications Power Automate email flows • "Ready for Review" (6 AM + 1 PM daily) when scan/storage check complete • "Migration Results" (7 AM daily) with completion status • Grouped by ITDistro
Error Handling Detailed error logging with CSV attachments • Partial migration recovery (MigratedWithErrors) • Automatic retry on transient failures
Migration Features Incremental/delta migrations • SCA permission grants on target sites • Claim-based item locking to prevent conflicts

Empty Folder Cleanup (Line 395)

Safety Features:

  • Triple-check verification before each deletion
  • Race condition protection (items appearing between checks)
  • Bottom-up deletion (deepest folders first)
  • Full logging in DeletionReport.csv

Weeknight Window (Lines 1789-1796) — Small items <10GB:

# Blocked hours: 4 AM (04:00) to 5 PM (16:59) local time
# Allowed hours: 5 PM (17:00) to 4 AM (03:59) local time
if ($hour -ge 4 -and $hour -lt 17) {
    # BLOCKED - business hours
    return $false
}
# ALLOWED - evening/night
return $true
To change business hours block:
Line 1792: Change 4 (start of block) and 17 (end of block)
Example: $hour -ge 6 -and $hour -lt 18 = 6AM-6PM blocked

Scheduling Logic

Size-Based Restrictions: Large migrations (≥10GB) are restricted to weekends and holidays to minimize impact on users. Use ANYTIME timezone for items that should run 24/7.

powershell.exe -NoProfile -ExecutionPolicy Bypass -File "F:\Scripts\Invoke-UNCStorageScan-v2.ps1" -AutoRun

Target Resolution (every 5 min):

powershell.exe -NoProfile -ExecutionPolicy Bypass -File "F:\Scripts\Update-MigrationTargets.v2.ps1" -AutoRun

powershell.exe -NoProfile -ExecutionPolicy Bypass -File "F:\Scripts\CommonDriveMigration.v2.ps1" -AutoRun -MaxItems 1 -Continuous -UseScheduling

  • -Continuous = Loop until MaxRuntime or queue empty
  • -UseScheduling = Respect timezone windows (5PM-4AM local)

powershell.exe -NoProfile -ExecutionPolicy Bypass -File "F:\Scripts\New-MigrationDashboard.ps1"

Troubleshooting

Task Won't Start

# Check task status
Get-ScheduledTask -TaskName "CommonDrive-Migration-AutoRun" | Select-Object State, LastRunTime, LastTaskResult

# View recent task history
Get-WinEvent -LogName "Microsoft-Windows-TaskScheduler/Operational" -MaxEvents 20 | 
    Where-Object { $_.Message -like "*CommonDrive*" }

Certificate Issues

# Verify certificate exists (replace with your thumbprint)
$thumbprint = "<your-cert-thumbprint>"
Get-ChildItem Cert:\LocalMachine\My | Where-Object { $_.Thumbprint -eq $thumbprint }

# Check certificate expiration
Get-ChildItem Cert:\LocalMachine\My | Where-Object { $_.Thumbprint -eq $thumbprint } | 
    Select-Object Subject, NotAfter, @{N='DaysUntilExpiry';E={($_.NotAfter - (Get-Date)).Days}}

Security Considerations

  • Credential File: SPMTCred.xml contains encrypted credentials. Protect with NTFS permissions.
  • Certificate: Only grant necessary accounts access to the private key.
  • Service Account: Use a dedicated service account with minimal required permissions.
  • Source Deletion: The script deletes source files after successful migration.
⚠️ WARNING: $EnableIncrementalDelete = $true and $EnableEmptyFolderCleanup = $true will permanently delete files and folders from the source after successful migration. Test on a small, non-critical folder first before running bulk migrations.
